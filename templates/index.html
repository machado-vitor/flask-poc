Jenkins CI/CD Pipeline Demo

Flask application deployed with Jenkins to Kubernetes

CI/CD Pipeline Workflow

This application demonstrates a full CI/CD pipeline:

  1. Code changes are pushed to Git repository
  2. Jenkins detects changes and triggers the pipeline
  3. Tests are run to validate code quality
  4. Docker image is built and tagged
  5. Application is deployed to Kubernetes
Kubernetes Deployment

This application is running on Kubernetes with:

  • Deployment with replica management
  • Service for internal networking
  • Ingress for external access (if configured)
  • ConfigMaps and Secrets for configuration
Flask Application

REST API Endpoints:

  • /api/hello - Basic greeting
  • /api/status - Service health check
  • /api/info - Server information
